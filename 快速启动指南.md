# litemall 快速启动指南

## 📋 脚本说明

项目根目录下提供了以下自动化脚本：

### 🔧 核心脚本

| 脚本名称 | 说明 | 需要管理员权限 |
|---------|------|--------------|
| `一键初始化.bat` | 完整初始化环境（配置环境变量、启动MySQL、导入数据库） | ✅ 是 |
| `启动系统.bat` | 启动后端和前端服务（可选择性启动） | ❌ 否 |
| `系统诊断.bat` | 检查系统状态和环境配置 | ❌ 否 |
| `配置环境变量.bat` | 单独配置Java和Maven环境变量 | ✅ 是 |

---

## 🚀 快速开始（3步启动）

### 方式一：全新安装（推荐）

如果你还没有配置过环境：

```batch
# 第1步：以管理员身份运行一键初始化
右键点击 "一键初始化.bat" → 选择 "以管理员身份运行"

# 第2步：关闭并重新打开命令行窗口（让环境变量生效）

# 第3步：启动系统
双击运行 "启动系统.bat"
```

### 方式二：已有环境（快速启动）

如果环境已经配置好：

```batch
# 直接运行启动脚本
双击运行 "启动系统.bat"
```

---

## 📖 详细步骤说明

### 步骤1: 运行一键初始化

**操作**：右键点击 `一键初始化.bat` → 选择"以管理员身份运行"

**执行内容**：
1. ✅ 配置JAVA_HOME环境变量
2. ✅ 配置MAVEN_HOME环境变量
3. ✅ 添加Java和Maven到PATH
4. ✅ 启动MySQL80服务
5. ✅ 导入litemall数据库
6. ✅ 创建litemall数据库用户（用户名：litemall，密码：litemall123456）

**注意事项**：
- 如果MySQL需要root密码，脚本会提示输入
- 如果数据库已存在，会自动跳过导入
- 初始化完成后需要重新打开命令行窗口

### 步骤2: 检查系统状态（可选）

**操作**：双击运行 `系统诊断.bat`

**检查内容**：
- Java环境（JDK是否安装）
- Maven环境
- 系统环境变量配置
- MySQL服务状态
- MySQL连接测试
- litemall数据库状态
- 项目文件完整性

### 步骤3: 启动系统

**操作**：双击运行 `启动系统.bat`

**启动选项**：
```
[1] 仅启动后端服务 (端口 8080)
[2] 仅启动管理后台前端 (端口 9527)
[3] 仅启动Vue移动端 (端口 6255)
[4] 全部启动（推荐）
[5] 后端 + 管理后台
```

## ⚠️ 常见问题

### Q1: 提示"需要管理员权限"
**解决**：右键点击脚本，选择"以管理员身份运行"

### Q2: MySQL服务启动失败
**解决**：
```batch
# 检查MySQL服务名称
sc query | findstr MySQL

# 尝试启动（可能服务名不是MySQL80）
net start MySQL80
# 或
net start MySQL
```

### Q3: 环境变量不生效
**解决**：
1. 关闭所有命令行窗口
2. 重新打开命令行窗口
3. 运行 `系统诊断.bat` 检查

### Q4: 数据库连接失败
**解决**：
```batch
# 检查MySQL服务是否运行
sc query MySQL80

# 测试MySQL连接
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
```

### Q5: 端口被占用
**解决**：
```batch
# 查看端口占用情况
netstat -ano | findstr ":8080"
netstat -ano | findstr ":9527"

# 结束占用进程
taskkill /F /PID <进程ID>
```

### Q6: JAR文件不存在
**解决**：
```batch
# 重新构建项目
cd D:\Code\litemall-master
mvn clean package -DskipTests
```

## 🔧 手动操作（可选）

### 手动配置环境变量

如果脚本无法运行，可以手动配置：

1. 打开"系统属性" → "高级" → "环境变量"
2. 在"用户变量"中添加：
   - 变量名：`JAVA_HOME`
   - 变量值：`D:\DevTools\jdk-11`

3. 在"用户变量"中添加：
   - 变量名：`MAVEN_HOME`
   - 变量值：`D:\DevTools\apache-maven-3.9.6`

4. 编辑"用户变量"中的"Path"：
   - 添加：`%JAVA_HOME%\bin`
   - 添加：`%MAVEN_HOME%\bin`

### 手动导入数据库

```batch
cd D:\Code\litemall-master\litemall-db\sql

# 1. 创建数据库
mysql -u root < litemall_schema.sql

# 2. 创建表
mysql -u root litemall < litemall_table.sql

# 3. 导入数据
mysql -u root litemall < litemall_data.sql
```
